What do I want out of classes in quartz?:

I want them to be like java classes, but more public
I'd also like writing getter/setters to be easier

{r} -> lets values be read but not modified
{w} -> lets values be modified but not read
{r, w} || {w, r} -> lets values be read and written

Example:

class Student {

    uint id {r}; 
    string name {r, w};

    static uint nextId = 1;

    Student(string name){

        this.id = nextId;
        this.name = name;

        nextId += 1;

    }

}

Extra Idea:

allow custom functions to be ran when r or w is triggered

class EvensOnly : throws Exception {

    uint number {
        r(() => {
            println("EvensOnly member has been read!!");
        }), 
        w(validateRule)
    };

    EvensOnly(uint number) {
        this.number = number;
    }

    void validateRule(uint number) {

        if(number % 2 != 0){
            throw new Exception("Not an even number! EWWWWWW!!!!");
        }

    }

}
